动态规划总结

#Dynamic Programming

##解题思路

解决动态问题的核心是写出动态转移方程，那怎么写出动态转移方程呢。在做题的过程中我发现，一旦确定了dp数组的维度会有益于写出动态转移方程。至于判断是1维还是2维，看有几个增长的变量就行了。1维动态转移方程非常简单，这里就不讨论了。写2维动态转移方程的时候，可以根据2个for循环的思路来写。也可以在草稿纸上画一张表，然后根据已知的求第dp(i+1,j+1)项。还有一点值得注意的是，如果dp(i+1,j+1)仅仅取决于一个值，那么可以用1维数组记录我们需要的值，尽管具有两个增长变量。

ugly number|求第n个2，3，5构成的数字|这个递归表达式还是非常巧妙的，和之前的都有点不一样|
Different Ways to Add Parentheses|Given a string of numbers and operators, return all possible results from computing all the different possible ways to group numbers and operators.|动态规划的思路是|

1 Different Ways to Add Parentheses
其实拿到一个问题的时候，我们不会很明显的知道是否可以采用动态规划，这个时候，我们先走一小步，看看如果是人，会怎么做。以Different Ways to add Parentheses, 为例，这一小步就是，选取两个相邻的数，然后结合成一个数。然后问题的规模就变小了，因为可以有很多的选择，这时候很明显就是DFS的思路。但是我们发现这样程序开销很大，每次要产生新数组，而且存在重复计算。
2 这时候我们考虑递归的思路，就是从小往大程序不好写的时候，这时候试试递归，发现采用递归的框架，程序就好写多了，同时采用动态规划记录中间结果，这样就解决了重复问题。

take away: 从小到大解决问题行不通的时候，尝试递归。

#DFS
DFS题目有个很明显的规律，题目就是要求一个满足条件的item,方法很简单，就是采用dfs一步一步的积累这个item,item满足条件则返回，不满足则该dfs终止。通常采用递归。

1) 这种问题要注意的是，dfs怎么归零。通常会采用如下的数据结构，最好是用引用，如果不能用引用，看是不是能采用一个used数组。

2) 去重问题，排序，然后使得nums[i]!=nums[i-1];

总之dfs通常来讲有3要素：

  1. result, item 每次dfs要build的东西，符合条件将item push 到 result中。
  2. change variable, 因为每次dfs的时候，问题的规模都会变得更小。
  3. condition variable, 随着item的积累，push的条件可能会改变。
题目名称 题目描述 解题思路
Generate Parentheses given n = 3, a solution set is:”((()))”, “(()())”, “(())()”, “()(())”, “()()()” trivial
combination sum1 & 2 For example, given candidate set 2,3,6,7 and target 7, A solution set is: [7] , [2, 2, 3] 1)result,item = vector 2)change variable = start index 3) target value = value - a[i]

Missing Number Given an array containing n distinct numbers taken from 0, 1, 2, …, n, find the one that is missing from the array. xor

Multiply Strings 这里有个很好的技巧是：“我们仍然是从低位到高位对每一位进行计算，假设第一个数长度是n，第二个数长度是m，我们知道结果长度为m+n或者m+n-1（没有进位的情况）。对于某一位i，要计算这个位上的数字，我们需要对所有能组合出这一位结果的位进行乘法，即第1位和第i位，第2位和第i-1位，… ，然后累加起来，最后我们取个位上的数值，然后剩下的作为进位放到下一轮循环中。”
